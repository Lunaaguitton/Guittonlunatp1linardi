Si vous n'apportez pas de modifications aux fichiers binaires et que vous souhaitez simplement les ajouter à votre dépôt Git, c'est la procédure correcte. Cependant, comme ils sont binaires, Git ne montrera pas de différences ou de modifications dans ces fichiers de la manière dont il le ferait pour les fichiers texte.
Si vous avez déjà exécuté ces commandes et que vous obtenez toujours "nothing to commit, working tree clean", cela signifie que Git considère que rien n'a changé dans ces fichiers binaires depuis le dernier commit. C'est tout à fait normal pour des fichiers binaires immuables.
En résumé, si vous n'avez pas apporté de modifications aux fichiers binaires, les commandes que vous avez utilisées pour les ajouter et les commettre sont correctes. Git ne montrera pas de différences visibles pour les fichiers binaires, mais il les suivra et les conservera dans l'historique des versions de votre dépôt.
